Codex に渡す依頼テンプレ

Claude Code が崩れた時に、Codex に何をどう渡せばぶれにくいかを、 そのままコピペできる文面つきでまとめたページです。

結論

Claude Code が崩れた時は、Codex に渡す用に「やること・完成条件・今の異常」を短く固定して投げる のが一番ぶれにくいです。Codex はコードを書いたり、既存コードを読んだり、 バグ修正やレビュー用変更提案を進める用途に向いています。

要約

  • Codex は ローカルやクラウドでコード作業を進めるコーディングエージェント として案内されています。
  • 最初の入力は、長い背景説明より 「今回やること」「ゴール」「壊れている点」 の3点に絞る方が通しやすいです。
  • Codex は ChatGPT のプランに含まれる案内があります。
  • なので今回は、原因究明を長く書くより 再現条件つきの修正依頼 にした方がいいです。

Codex にそのまま投げる文面

最小修正の意図がぶれにくい形に整理しています。

いまのアプリ実装が壊れています。原因調査と最小修正をしてください。

## 今回やってほしいこと
- 現在のエラー原因を特定する
- 最小修正で動作を復旧する
- 修正内容と原因を短く説明する

## 現状
- 期待動作:
  - 推しごとの一覧が表示される
  - ログ画面で記録追加できる
  - メンカラが正しく反映される
- 実際:
  - Claude Codeで途中から意味不明な状態になった
  - 画面や状態管理のどこかが崩れている

## 優先順位
1. コンパイル/実行を通す
2. 画面崩れや状態崩れを直す
3. 不要な大改修はしない

## 制約
- 既存UIの黒ベース世界観は維持
- 推しの色は以下を基準にする
  - はうき: #4A86B8
  - めーとる: #F3F0EB
  - M: #E8A3BE
  - JOKER: #B88AD8
- まずは最小修正優先
- 足りない箇所は仮実装でよい

## 完成条件
- 実行できる
- 推し一覧が表示される
- 色設定が反映される
- 何を直したか箇条書きで報告する

コード全体を確認して、原因→修正→確認手順の順で進めてください。
