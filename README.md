# New Project

## 1. プロジェクト名（仮）

Mini Workflow Hub

## 2. 目的

小さく作ってすぐ使える作業支援ツールを1つ完成させる。
最初から大きな機能を狙わず、短いサイクルで形にしながら開発の流れを固める。

## 3. 何を作るか

このプロジェクトでは、まず以下の3案を用途候補として考える。

### 案1: 個人用タスク整理Web

- 今日やること、進行中、完了をシンプルに管理する小さなWebツール

### 案2: 作業メモ整理アプリ

- 思いついたメモ、調べたこと、次にやることを1か所にまとめる軽量アプリ

### 案3: 開発練習用の実験プロジェクト

- UI、Git、設計、ツール導入を試しながら学ぶための検証用プロジェクト

現時点で一番向いている案は、**案1: 個人用タスク整理Web**。
理由は、作るものが明確で小さく始めやすく、画面・データ・操作の基本を一通り練習できるため。

## 4. 使いそうな技術候補

- HTML / CSS / JavaScript
- TypeScript
- Vite
- React
- 必要に応じて localStorage

最初は複雑な構成にせず、フロントエンドだけで完結する形を優先する。

1. プロジェクト名と用途をこのREADMEの内容で仮決定する
2. 最小画面として「タスク一覧 + 追加フォーム」の1画面に絞る
3. その後、必要なら最小構成のWeb雛形を作る
