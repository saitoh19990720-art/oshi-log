Claude Code運用のルールと仕事フローを確認する。迷ったときに開く。

---

## 役割分担（固定）

```
しずく = 設計・世界観・最終判断
Claude Code = 実装・修正・整形・反復
```

---

## 運用ルール

- 1案件1スレッド
- 1回1目的
- まず最小構成
- 修正は優先順位つきで投げる
- 完成したら /impl-log でテンプレ化
- 比較・調査は月1だけ（実行フェーズで使う）

---

## 仕事フロー

```
1. 題材を1つ決める
2. Figmaで1画面決める
3. Claude Codeに実装させる（/template-lp）
4. 自分が余白と文を直す（/lp-check）
5. Claude Codeで微修正（/lp-revise）
6. 公開
7. 提案文へ変換（/proposal）
```

---

## やること優先順位

| 軸 | 内容 |
|---|---|
| ① LP制作 | 1ページ完結・案件化しやすい |
| ② 小規模Web | サービス紹介・プロフィール |
| ③ Figma→実装 | しずくの強み候補 |
| ④ 既存ページ改善 | 提案しやすい・受注率高い |
| ⑤ テンプレ商品化 | 将来の資産 |

---

## やらないこと

- 何を作るか未定のまま実装
- 完成前にツール比較へ逃げる
- Figmaなしでデザインを丸投げ
- アイデア探索だけで延々会話
