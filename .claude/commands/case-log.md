案件応募ログを記録する。応募前チェック・ログ出力・返信対応テンプレを提供する。

## 使い方
「case-log」と入力し、状況（応募前/返信きた/受注）を伝える。

---

## 応募前チェック
以下を確認してOKなら応募に進む：
- LP / Webページ / Figma / HTML/CSS に近いか
- 通話前提すぎないか
- 納期が無理すぎないか

**見送る基準:** 通話・常駐が強い / 要件が広すぎる / LP制作から離れすぎる

---

## 応募ログ出力フォーマット
```
## YYYY-MM-DD
案件名：
何に応募したか：
差し替えた点：
返事待ち：
次やる（最初の1手）：
```

---

## 返信が来た時
**基本返信:**
```
ご連絡ありがとうございます。内容、確認しました。
このまま進める場合、まずは目的・必要な成果物・納期の3点を確認できると進めやすいです。
分かる範囲で共有いただければ整理して進行案をまとめます。よろしくお願いいたします。
```

**通話なしをやわらかく伝える:**
```
私は基本的に、テキストベースで要点を整理しながら進める形を中心にしています。
まずは目的・必要なページ内容・納期を送っていただければ、進め方を整理してお返しします。
```

---

```

目的：

最初の確認事項：目的 / 必要な内容 / 参考イメージ / 素材の有無
```

**クライアントへの最初の文:**
```
ご依頼ありがとうございます。
まずは目的・必要な内容・納期を確認させてください。
分かる範囲で共有いただければ、進め方を整理してお返しします。
```
