n8nワークフローを設計する。最小構成・イベント駆動・常時起動なし。

## 使い方

「何をトリガーに」「何をして」「どこに保存・通知するか」を伝える。

---

## 設計原則

```
常時起動 ❌ → イベント駆動 ✅
1フロー = 1用途（詰め込まない）
失敗したらループではなく停止 + 通知
```

---

## 手順

1. **入力（トリガー）** を確定 — Webhook / RSS / スケジュール / 手動
2. **処理** を最小で設計 — ノード数を絞る
3. **出力先** を確定 — Discord / Obsidian / ファイル保存 / HTTP送信
4. ノード構成をテキストで出す（フロー図）
5. 必要なら JSON の骨格を出す

---

## 出力フォーマット（フロー図）

```
[トリガー] → [処理1] → [処理2（必要時のみ）] → [出力]
```

例:
```
[RSS Feed] → [フィルタ: キーワードあり] → [Discord Webhook通知]
```

---

## 認証情報の扱い

- webhook URL・APIキーは必ず `$env.VAR_NAME` で参照
- JSONに直書きしない
- 変数名はすべて大文字スネーク（例: `DISCORD_WEBHOOK_URL`）

---

## しずく環境の既存変数

| 変数名 | 用途 |
|---|---|
| `DISCORD_WEBHOOK_URL` | #案件通知 |
| `DISCORD_WEBHOOK_URL_INFO` | #情報収集 |

---

## 制約

- 1フローに複数MCPを詰め込まない
- 「とりあえず全部つなぐ」はしない
- 設計が確定してから JSON を出す（先に JSON を出さない）
