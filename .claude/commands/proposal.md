案件文を受け取り、CrowdWorks / Lancers 用の応募文を生成する。

## 手順
1. 案件文を読み、**案件タイプ**を判定する
2. 対応するテンプレを選んで差し替え3項目を埋める
3. 完成版・差し替えメモ・送信前チェックを返す

---

## 案件タイプ別テンプレ

### A. LP制作（メイン）
```
こんにちは。[案件名・業種の一文]、LP制作の目的や導線設計まで含めてお手伝いできると感じご連絡しました。

Figmaでの設計からClaude Codeを活用した実装まで一気通貫で進めています。非同期・テキスト中心で整理しながら形にする進め方が得意です。

[実績1文]途中共有もテキストベースで行い、認識のズレを減らしながら進行します。[納期・稼働条件]よろしくお願いいたします。
```

### B. Figma設計・ワイヤーのみ
```
こんにちは。[案件名の一文]、Figmaでの画面設計からお手伝いできると感じご連絡しました。

構成整理・情報設計・UI設計まで、テキストベースで認識を揃えながら進めます。[実績1文]納期や調整は柔軟に対応できます。よろしくお願いいたします。
```

### C. HTML/CSSコーディング
```
こんにちは。[案件名の一文]、HTML/CSSのコーディングでお力になれると感じご連絡しました。

デザインカンプからの実装・レスポンシブ対応・既存サイトへの組み込みに対応しています。[実績1文]非同期・テキスト中心で進行します。[納期]よろしくお願いいたします。
```

### D. ページ修正・軽作業
```
こんにちは。[案件名の一文]、修正・調整のご対応ができると感じご連絡しました。

HTML/CSS/JSの修正・追記・デザイン調整に対応しています。テキストベースで要件を確認しながら、最小差分で対応します。[納期]よろしくお願いいたします。
```

---

## 差し替え3項目（共通）
1. **案件名 / 業種** — 冒頭1文に自然に入れる
2. **近い実績** — あれば1文。なければ省く（`[実績1文]`を削除）
3. **納期 / 稼働条件** — 案件の要件に合わせる

---

## 短縮版（150字以内・迷ったらこれ）
```
案件を拝見し、[タイプ]でお力になれると感じご連絡しました。Figma設計からHTML/CSS実装まで一気通貫で進めています。非同期・テキスト中心で整理しながら形にできます。
```

---

## 出力形式
1. **案件タイプ判定**（A/B/C/D）
2. **完成版**（コピペ用）
3. **差し替えメモ**（何を入れたか確認用）
4. **送信前チェック**（通話前提か・200字超えか・盛っていないか）

---

## 制約
- 盛らない・嘘を書かない
- 通話・常駐が強い案件には「⚠ 非同期進行が難しい案件です」と警告する
- 要件が広すぎる案件には「⚠ 範囲が不明確です。応募前に要件確認を推奨」と警告する
