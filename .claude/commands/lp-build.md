LPを段階的に設計・実装する。一気に全部作らず、フェーズを区切って進める。

## 使い方

依頼内容（クライアント業種・要件・参考があれば）を貼るだけ。
Figmaデータがあれば「Figmaあり」と一言添える。

---

## 進め方（4フェーズ固定）

### Phase 1 — 構成定義（しずくのYes/No待ち）
出すもの:
- セクション構成案（6〜8セクション）
- ファーストビューのコピー案（2〜3パターン）
- 世界観・配色方針（1行）

### Phase 2 — デザイントークン確定
出すもの:
- CSS変数（カラー・フォント・間隔）
- 使用フォント（Google Fonts / SIL OFL商用可のみ）
- ここでしずくのOKをもらってから実装へ

### Phase 3 — HTML/CSS実装
- セクション単位で実装（一括で全部作らない）
- 1セクション = 1コミットを基本とする
- SP対応（375px）は各セクション実装時に同時に行う

### Phase 4 — 仕上げ・納品準備
- 全体通し確認（PC1280 / SP375）
- アニメーション調整（必要なら）
- `/delivery-check` を実行して納品

---

## 制約

- 「全部一気に作って」と言われても Phase 1 の確認を先に取る
- 世界観の判断は必ずしずくに委ねる（自分で決めない）
- フォントは著作権セーフ（Google Fonts中心）のみ
- デモURLは必要になったら Vercel にデプロイ

## デフォルトデザイントークン（水色透明系）

```css
:root {
  --bg: #e8f2f6;
  --panel: rgba(255, 255, 255, 0.85);
  --panel-border: rgba(100, 160, 180, 0.3);
  --text: #1a2a32;
  --muted: #4a7080;
  --accent: #3a9ab8;
  --accent-soft: rgba(58, 154, 184, 0.12);
  --pink: #b07888;
  --lavender: #8878b0;
}
```
（案件の世界観に合わせて変更する）
