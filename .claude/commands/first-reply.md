受注・問い合わせへの初回返信文を出す。状況に合わせて3パターン。

## 使い方

「受注した」「問い合わせが来た」「断りたい」と一言添えるだけ。

---

## パターン① 問い合わせへの返信（受注前）

```
はじめまして、しずくと申します。
ご連絡いただきありがとうございます。

内容を確認しました。対応可能です。

いくつか確認させてください。

① 完成イメージや参考サイト・画像はありますか？
② ご希望の納期はいつ頃でしょうか？
③ 公開方法のご希望はありますか？（Vercel公開 / ファイル納品 / その他）

上記が揃いましたら概算とスケジュールをご案内します。
よろしくお願いいたします。
```

---

## パターン② 受注確定後の最初のメッセージ

```
受注いただきありがとうございます。しずくです。

以下の流れで進めます。

【進め方】
1. 要件確認（このメッセージで完了させます）
2. 設計（Figma or 構成案）→ 確認
3. 実装 → 確認
4. 修正 → 納品

【確認事項】
以下を教えていただけますか？

・参考サイト or 参考画像（あれば）
・使用する色・フォントの指定（あれば）
・盛り込みたい内容・セクション
・公開方法（Vercelでの公開 or ファイル納品）
・修正回数の上限（目安：2回まで）

不明な点があれば何でも聞いてください。
よろしくお願いいたします。
```

---

## パターン③ お断り（対応不可）

```
ご連絡いただきありがとうございます。

誠に恐れ入りますが、今回は対応が難しい状況です。
（理由：[納期が合わない / 専門外の技術を含む / スケジュール満員]）

またご縁がありましたらよろしくお願いいたします。
```

---

## 制約

- 200字以内に収める
- 「丁寧に対応します」「精一杯努めます」系は使わない
- 確認事項は箇条書き・最大4点
- 断りは理由を1つだけ明示して短く切る
