# /lp-prompt — LP実装の完成プロンプト

案件受注時・自己LP制作時に、要件と本文を埋めた状態で Claude Code（または別セッション）に渡すための雛形。
本スキルは **雛形を出力するだけ** で、実装はしない。出力された内容を別途コピペして使う。

## 使い方

```
/lp-prompt <案件テーマ>
```

例: `/lp-prompt 美容サロンLP` / `/lp-prompt 推し活記録アプリLP`

テーマに応じて必要セクションと本文トーンを差し替えた雛形を出す。

## 出力テンプレ（要件ブロック）

```md
ポートフォリオ用/案件用のLPを1ページ構成で作ってください。

あなたは実装担当です。私は設計と最終判断を担当します。
完璧な完成版より、まず「動く・崩れにくい・後で直しやすい」ものを優先してください。
不足情報があっても止まらず、合理的に補完して進めてください。
推測した部分は「推測」と明記してください。

# 目的
<1〜2行でターゲットと伝えたい価値を明示>

# 想定読者
- <具体的ペルソナ3〜4本>

# トーン
- 透明感・静か・見やすい
- 営業っぽすぎない / 甘すぎない / 堅すぎない

# デザイン傾向
- 水色・白・灰ベース（案件により差替）
- 余白重視・スマホで崩れない
- 過剰アニメーション不要・読みやすさ優先

# 必要セクション
- ファーストビュー / できること / 制作の流れ / 向いている依頼 / サンプル / 問い合わせ
（案件に応じ削除/追加）

# ページ内本文案
<各セクションの本文たたき台を埋める>

# 実装条件
- HTML/CSS最小構成で完成
- スマホ優先
- React/Next.js に移しやすい素直な構造
- 日本語フォント前提
- 過剰装飾禁止・余白と文字組みで整える

# 出力形式
1. 実装方針
2. ファイル構成
3. HTML
4. CSS
5. 補足
6. 次に確認すべき点

# 注意
- 70点でも公開できる状態を優先
- 1回で盛り込みすぎず、直しやすい構造を保つ
- 不明点で止まらず自然に補完
```

## 配色メモ（水色透明系デフォルト）

要件ブロックに追加できる配色指定のテンプレ。

```md
# 配色メモ
- ベース背景: ごく薄い青みホワイト
- セクション背景: 白または薄いグレー
- メインカラー: やわらかい水色
- サブカラー: 淡いグレー・落ち着いた青
- 文字色: 黒ではなく青みのあるダークグレー
- 強いコントラストは避ける / 彩度は高すぎない
- CTAボタン: 水色ベース・白文字 or 濃いグレー文字
- 全体として「冷たすぎない静かな透明感」を優先

# UIメモ
- Heroは中央寄せもOK・本文は読みやすさ優先
- カードは角丸を少し・影は強くしない
- 境界線は薄く
- セクション間の余白は広め
- 文字サイズ差をつけすぎない
- 装飾で埋めず、余白で整える
```

## レビュー観点（投げた後の見方）

**まず見る:**
- ファーストビューが重すぎないか
- スマホで縦に読めるか
- CTAが押し売りっぽくないか
- 余白が詰まりすぎていないか

**直す優先順:**
1. 文の長さ
2. セクション順
3. 余白
4. 配色
5. 細かい装飾

## LP 確認チェックリスト（初稿が出た後）

```md
## 構造
- ファーストビューで何の人かわかるか
- セクション順が自然か
- CTAまで無理なく読めるか

## 見た目
- 余白が詰まっていないか
- 水色が強すぎないか
- SaaSテンプレ感が強すぎないか
- 可愛すぎず、静かに整っているか

## スマホ
- 縦に読んで苦しくないか
- ボタンが押しやすいか
- 見出しが長すぎないか

## 文
- 営業っぽすぎないか
- 抽象語が多すぎないか
- 自分の仕事がちゃんと伝わるか
```

## 2回目の修正プロンプト（初稿修正用）

```md
ありがとう。全体の方向は良いです。次は以下を修正してください。

## 修正したい点
- 余白をもう少し広くしたい
- SaaSテンプレっぽさを減らしたい
- 文字色を少しやわらかくしたい
- ファーストビューの営業感を少し弱めたい
- スマホでの見出しサイズを少し整えたい

## 優先順位
1. 余白
2. ヒーローの空気感
3. 配色
4. スマホの文字サイズ

## 方針
- 構造は大きく崩さない
- 必要部分を中心に直す
- 変更点がわかるようにする
- 完全刷新ではなく、静かに改善する方向で進める

出力形式:
1. 変更方針
2. 変更後コード
3. どこをどう直したか
```

## 注意

- 本スキルは**雛形出力のみ**。実装・デプロイは別途 `/lp-build` に繋ぐ
- 母艦 oshi-log / aqua-festa と重複するテーマ（ポートフォリオ / 推し活イベント）は新規に作らず既存を活用
