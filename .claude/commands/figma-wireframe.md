# /figma-wireframe — Figmaワイヤー指示文

LP/Webページ制作時、実装に入る前に Figma で配置だけ固める段階で使う指示雛形。
装飾を先に決めず、**配置と情報構造だけ** を確定させるのが目的。

## 使い方

```
/figma-wireframe <テーマ>
```

## 出力テンプレ

```md
Figmaで下記LPのワイヤー（下書き）を置いてください。
装飾・色・フォント沼には入らず、配置と情報順だけ固めます。

# 目的
<1〜2行>

# アートボード
- PC: 1440 × セクション分の高さ
- SP: 375 × セクション分の高さ
- 両方同じFigmaファイル内の別ページ推奨

# セクション構成（順番固定）
1. Hero — h1 + リード + CTA
2. できること / サービス概要 — 3〜4項目
3. 制作の流れ — 4〜5ステップ
4. 向いている依頼 — 3〜5項目
5. サンプル / 実績 — カード3〜4枚
6. 問い合わせ導線 — CTA + 補足

# 各セクションに置くもの（装飾なしでOK）
- 見出し（太字のプレーンテキスト）
- 説明文（ダミーで可）
- ボタン（角丸四角 + テキストラベル）
- 画像プレースホルダ（グレー四角 + "img"）
- カード（薄いグレー枠 + 角丸 + 中身テキスト）

# 配置ルール
- 1セクション = 1 Auto Layout フレーム
- 横幅: PC 最大1040px / SP 100% (横パディング16px)
- セクション間の余白: 72-96px
- 要素間の余白: 16-24px
- 見出しとリードの間は大きめ
- CTAは画面内で迷わないサイズ

# やらないこと
- 色指定（後で配色メモで流し込む）
- フォント選定（後で決める）
- 写真素材の選定（プレースホルダで進める）
- アニメーション・マイクロインタラクション
- コンポーネント化の作り込み（最初は素のフレームでOK）

# 完了の定義
- PC/SPの両方で、縦スクロール1回で全体が見える
- 見出し → リード → CTA の順に視線が自然に降りる
- 各セクションが視覚的に区切られている
- コピペで実装担当に投げられる情報量がある
```

## 次の工程との接続

1. `/figma-wireframe` でワイヤー完成
2. `/lp-prompt` で要件+本文+配色を言語化
3. Claude Code に流して HTML/CSS 実装
4. `/delivery-check` で最終確認
5. Vercel デプロイ → Works 枠に追加

## 注意

- ワイヤー段階で**配色・フォントで悩まない**（Ne収束の罠を避ける）
- 装飾は最後。先に構造を確定させることが速さに直結する
- Figmaで2時間以上悩んだら、一度HTMLに落として現物で考え直す
