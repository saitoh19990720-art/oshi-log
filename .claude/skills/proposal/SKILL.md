# Proposal Skill

## 使い方
案件文をそのまま貼って「proposal を使います」と言うだけ。

## 出力形式
1. 完成版（そのままコピペできる文）
2. 差し替えた箇所の確認メモ（3項目）

## ベーステンプレ
```
こんにちは。[案件名/業種の一文]、LP制作の目的や導線設計まで含めてお手伝いできると感じ、ご連絡しました。

私は、Figmaでの設計からClaude Codeを活用した実装まで、一気通貫で進めています。LPやWebページを、非同期・テキスト中心で整理しながら形にする進め方が得意です。

[近い実績・経験があれば1文]途中共有もテキストベースで行い、認識のズレを減らしながら進行します。[納期・稼働条件]よろしくお願いいたします。
```

## 差し替え3項目
1. **案件名 / 業種** — 冒頭1文に自然に入れる
2. **近い実績・経験** — あれば1文。なければ省く
3. **納期 / 稼働条件** — 案件の要件に合わせる

## 短縮版（150字以内）
```
案件を拝見し、LP制作の目的や導線整理まで含めて対応できると感じご連絡しました。Figma設計からClaude Codeを活用した実装まで一気通貫で進めています。非同期・テキスト中心で整理しながら形にできます。
```

## 送信前チェック
- [ ] 案件理解の一文がある
- [ ] 非同期・テキスト中心が自然に入っている
- [ ] 長すぎない・盛っていない

## 制約
- 盛らない・嘘を書かない
- 通話前提の案件には使わない
- 200字を超えたら短縮版を使う
