# Claude Code運用ルール（高負荷制御）

> Claudeは強いから節約するのではなく「暴走させない」が正解。

**作成日：** 2026-04-17
**格納先：** `prompts/claude-code-rules.md`

---

## 🎯 結論

Claude Codeは自由に使うと暴走する。Codexより **強く制御するルール** が必要。

- 重い処理・長文・設計専用
- コスト増の原因は「考えすぎ・広げすぎ」
- 対策は **範囲固定・出力制限・段階実行**
- 「どこまでやるか」を毎回明示

---

## 1. 基本方針

- Claude Codeは **重い処理専用**
- 設計・長文・複雑処理のみ担当
- 1回の実行範囲を必ず制限する

---

## 2. 入力ルール

| ❌ NG | ✅ OK |
|---|---|
| 「全部やって」 | 目的を1行で固定 |
| 曖昧なゴール | 対象範囲を明示 |
| 無制限な思考要求 | 出力形式を指定 |
| 大量ファイル一括投入 | 制約条件を書く |

**例：**
```
目的：LPの構造だけ設計
範囲：1ページ分のみ
出力：見出し構造
制約：コード不要
```

---

## 3. 範囲固定ルール（最重要）

必ず書く：

- 「この範囲だけ」
- 「ここまでで止める」
- 「次はやらない」

**例：**
> 「HTML構造だけ作る。CSSはやらない」

---

## 4. 出力制御ルール

必ず付ける：

- 「必要最小限」
- 「冗長な説明不要」
- 「構造優先」
- 「結論→要点のみ」

---

## 5. 段階実行ルール

| ❌ NG | ✅ OK |
|---|---|
| 一発生成 | ① 設計 → ② 構造 → ③ 実装 → ④ 修正 |

> **1回 = 1段階**

---

## 6. 思考制御

Claudeは考えすぎるため制限：

- 「簡潔に」
- 「最大3パターン」
- 「1案でいい」

---

## 7. コンテキスト管理

- 案件ごとにスレ分離
- 長くなったら `/clear`
- 必要部分のみ渡す

---

## 8. 禁止事項

- 全自動生成
- 無制限リファクタ
- 全ファイル解析
- 「最適化して」だけの指示

---

## 9. 切り替え基準

Claudeを使う条件：

- 200行以上のコード
- 複数ファイル
- 設計が必要
- バグ原因分析

→ **それ以外はCodexへ**

---

## 10. 依頼テンプレ

```
目的：〇〇を設計したい
範囲：この部分のみ
出力：〇〇形式
制約：最小限・簡潔・ここまで
```

---

## 11. 一文ルール

> **広げるな、区切れ、止めろ**

---

## 12. 強制停止ルール

以下の場合は途中で止める：

- 出力が長くなりすぎた
- 想定外の方向に広がった
- 指示範囲を超えた

→ 「ここで停止」と明記させる

---

## 🫧 Codexとの違い

| モデル | 制御方針 |
|---|---|
| **Codex** | 軽い作業 → 削る（節約ルール） |
| **Claude** | 重い処理 → 制御する（暴走防止ルール） |

---

## 💎 セット運用の最適構造

```
ChatGPT     →  思考
Codex       →  軽作業
Claude Code →  本実装
```

> この構造が一番コスト安定（AI役割分担の基本）
