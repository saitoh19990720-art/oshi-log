# 初案件ランブック｜受注〜納品の全手順

更新: 2026-04-18

---

## フロー全体

```
応募 → 受注連絡 → ヒアリング → 制作 → 確認 → 納品 → /case-log
```

---

## STEP 1｜受注連絡が来たとき

確認する3点だけ：

- [ ] 何を作るか（LP / バナー / 修正 / Figmaデータ）
- [ ] 納品形式（HTML/CSS / Figmaファイル / Vercel URL）
- [ ] 希望納期

不明な場合はこのテンプレで確認：

```
ありがとうございます。進めるにあたって確認させてください。

・最終的な納品形式（HTML/CSSファイル / Vercel公開URL / Figmaデータ）
・ご希望の完成時期
・参考にしたいデザインや雰囲気（あれば）

確認後、着手スケジュールをお伝えします。
```

---

## STEP 2｜制作開始前にやること

- [ ] `デスクトップ/新しいフォルダー/` に案件フォルダ作成（例：`案件名-yyyymm/`）
- [ ] `index.html` + `style.css` を最小構成で作る
- [ ] ローカルでブラウザ確認できる状態にする

### Claude Codeへの最初の指示テンプレ

```
[案件概要]
作るもの：LP 1ページ（エステサロン / 界隈グッズ / etc）
対象ユーザー：○○（例：20〜30代女性）
参考：[URL or 界隈名]
納品形式：HTML/CSS + Vercel URL

[指示]
まず index.html と style.css の最小構成を作って。
セクション構成だけ先に確認させて。
```

---

## STEP 3｜制作中のチェックポイント

| タイミング | やること |
|---|---|
| セクション構成できたら | 先方に確認を取る（テキスト共有 or Vercel URL） |
| PC表示ができたら | SP（375px）確認 |
| 一通り完成したら | `/delivery-check` を呼ぶ |

### SP確認の最低ライン

- [ ] 文字が読める（14px以上）
- [ ] ボタンが押せるサイズ（44px以上）
- [ ] 横スクロールが出ていない
- [ ] 画像が縦長すぎて崩れていない

---

## STEP 4｜確認・修正ループ

修正依頼が来たとき：

```
[案件名] 修正：[内容を一言]
[該当箇所のスクリーンショットまたはURL]
```

をClaude Codeに貼るだけで対応できる。

**修正は3回以内が目安。** それ以上になる場合は：
- 要件が最初から曖昧だった
- 追加要望になっている

→ 追加費用の相談か、スコープを確認する。

---

## STEP 5｜納品

### 納品物チェック（`/delivery-check` と併用）

- [ ] Vercel URLが生きている
- [ ] SP表示確認済み
- [ ] コンソールエラーなし
- [ ] 依頼外の変更を入れていない

### 納品メッセージテンプレ

```
お待たせしました。完成しましたのでご確認ください。

URL：[Vercel URL]
スマホ・PCどちらでもご確認いただけます。

修正があればお知らせください。
```

---

## STEP 6｜完了後

- [ ] `/case-log` で記録（Obsidianに貼る）
- [ ] 実績化可能なら: ポートフォリオへの追加を検討
- [ ] 良かった点・詰まった点を一言メモ（`/daily-log`）

---

## よくある詰まりポイント

| 状況 | 対処 |
|---|---|
| Figmaが共有されない | 「閲覧権限のある共有リンクをいただけますか」と伝える |
| 要件が曖昧すぎる | STEP 1のテンプレで確認を入れる |
| 修正が増えすぎる | 「今回の修正でいったん確定にしましょう」と区切りを入れる |
| 急ぎ対応を求められる | 「当日〜翌日の納品は対応が難しく、[可能な日]での完成なら対応可能です」 |
