# Claude Code 運用OS

> 最小構成で「ズレずに回す」ための運用ルール

**作成日：** 2026-04-17
**格納先：** `prompts/claude-code-os.md`

---

## 🎯 結論

入れるべきは「AIを強くするもの」ではなく **「AIをズレさせないもの」**。
環境は **Claude Code + CLAUDE.md** の2つで足りる。

---

## 🧩 構成

```
Figma  ──▶  Claude Code  ──▶  VS Code
 設計         実装本体         差分確認
```

---

## ✅ 必須要素

| 要素 | 役割 |
|---|---|
| **Claude Code** | コード生成・修正・差分確認。Figma → 実装の本体 |
| **CLAUDE.md** | プロジェクトごとのAI設定。精度を決める要 |

---

## 📄 CLAUDE.md テンプレ（LP用）

````markdown
# Project Rules

## 目的
FigmaデザインをもとにLPを構築する

## ルール
- 勝手に構造を変えない
- Tailwind優先
- セクション単位で修正

## トリガー
- /plan で設計から入る
- 実装は小分けにする
````

---

## ⌨️ コマンド（3つだけ覚える）

| コマンド | 使いどき |
|---|---|
| `/plan` | いきなりコードを書かせない。設計を先に確認 |
| `/compact` | トークン節約。長くなったら叩く |
| `/clear` | 話題リセット。別タスクに切り替えるとき |

---
