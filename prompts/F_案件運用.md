# 案件運用セット｜応募〜受注〜納品

---

## 応募前チェックリスト

- [ ] LP / Webページ / Figma / HTML/CSS に近いか
- [ ] 通話前提すぎないか
- [ ] 納期が無理すぎないか
- [ ] 今週の勝ち条件に直結しているか

**見送る案件：** 通話・常駐が強い / 要件が広すぎる / LP制作から離れすぎる

## 3件応募の回し方

- 1件目：20〜30分（型を固める）
- 2件目：10〜15分（型を流用）
- 3件目：10〜15分（最小修正で出す）

**light の日：** 案件文を1本読むだけ or 冒頭1文だけ作るでOK

---

## 返信が来た時

**まずお礼して前に進める**
```
ご連絡ありがとうございます。内容、確認しました。
このまま進める場合、まずは目的・必要な成果物・納期の3点を確認できると進めやすいです。
分かる範囲で共有いただければ整理して進行案をまとめます。よろしくお願いいたします。
```

**通話なしをやわらかく伝える**
```
私は基本的に、テキストベースで要点を整理しながら進める形を中心にしています。
まずは目的・必要なページ内容・納期を送っていただければ、進め方を整理してお返しします。
```

---

## 案件を断る

```
ご連絡ありがとうございます。
今回は条件や進行の相性を踏まえ、見送らせていただければと思います。
またの機会がありましたら、よろしくお願いいたします。
```
