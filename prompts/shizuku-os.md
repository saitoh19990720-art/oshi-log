# Shizuku OS｜AI運用フロー

> AIは増やすと強くならない。**流れを固定すると強くなる。**

**作成日：** 2026-04-17
**格納先：** `prompts/shizuku-os.md`

---

## 🎯 結論

3つのAIは **1枚の流れ** に固定して迷いゼロにする。

- 構造は **ChatGPT → Claude Code → Codex** の一方向
- 分岐させず **役割固定** が最重要
- 判断はChatGPTで終了
- 実装はClaudeに集約
- Codexは "最後の調整だけ"

---

## 0. 全体構造

```
  思考         実装          軽作業
ChatGPT  →  Claude Code  →  Codex
```

---

## 1. ChatGPT（思考・設計）

| 項目 | 内容 |
|---|---|
| **役割** | 方向性決定／比較・優先順位／仮説・構造設計 |
| **入力** | 何を決めたいか |
| **出力** | 結論（1つ）＋ 要点（最大5） |
| **禁止** | 実装／長文生成／無限アイデア |
| **終了条件** | やることが **1つに決まる** |

---

## 2. Claude Code（本実装）

| 項目 | 内容 |
|---|---|
| **役割** | 設計を形にする／コード生成／構造構築／バグ修正 |
| **入力** | ChatGPTの結論 |
| **出力** | 動くもの／実装コード |
| **制約** | 範囲を必ず指定／段階実行 |
| **禁止** | 全自動生成／無制限処理 |
| **終了条件** | 一旦 **動く状態** |

---

## 3. Codex（軽作業）

| 項目 | 内容 |
|---|---|
| **役割** | 微修正／差分変更／小さいコード生成 |
| **入力** | 該当箇所のみ |
| **出力** | 最小コード |
| **制約** | 短く・軽く |
| **禁止** | 全体生成／長文処理 |
| **終了条件** | **整った状態** |

---

## 4. フロー（固定）

```
① ChatGPT  →  何をやるか決める
        ↓
② Claude Code  →  形にする
        ↓
③ Codex  →  整える
```

---

## 5. NGフロー（絶対やらない）

- ❌ ChatGPTで実装
- ❌ Claudeで全部やる
- ❌ Codexで大規模開発
- ❌ 行き来しまくる

---

## 6. 一文ルール

> **決めて → 作って → 整える**

---

```
  ↓              ↓

```
