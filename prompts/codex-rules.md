# Codex運用ルール（コスト最適化）

> Codexは「賢く使う」ではなく「雑に使えないように縛る」と強い。

**作成日：** 2026-04-17
**格納先：** `prompts/codex-rules.md`

---

## 🎯 結論

節約 = **トークン削減ルールを固定化すること**。
CLAUDE.md に "強制運用ルール" として入れるのが最適。

- 短く・分割・役割分担
- Codexは軽作業専用に制限
- 重い処理はClaudeへ逃がす
- 禁止事項を明文化して安定化

---

## 1. 基本方針

- Codexは **軽作業専用**
- 重い処理・長文生成は Claude Code へ回す
- 常に **最小トークン** で実行

---

## 2. 入力ルール

| ❌ NG | ✅ OK |
|---|---|
| 長文説明 | 必要部分のみ貼る |
| 不要な背景情報 | 目的を1行で書く |
| 全ファイル貼り付け | 出力形式を指定する |
| 曖昧な指示 | — |

**例：**
```
目的：ボタンのhoverだけ修正
出力：CSSのみ
```

---

## 3. タスク分割ルール（最重要）

| ❌ NG | ✅ OK |
|---|---|
| 「LP全部作って」 | ① 構造作成 → ② HTML → ③ CSS → ④ 修正 |

> **1タスク = 1目的**

---

## 4. 出力制御ルール

必ず指示する：

- 「最小コードで」
- 「説明不要」
- 「差分のみ」
- 「変更箇所のみ」

---

## 5. コンテキスト制御

- 会話は案件ごとに分ける
- 長くなったら `/clear`
- 必要なファイルだけ渡す

---

## 6. モデル役割分担

| モデル | 役割 |
|---|---|
| **ChatGPT** | 思考・設計 |
| **Codex** | 軽い修正・コード生成 |
| **Claude Code** | 本実装・重い処理 |

---

## 7. 禁止事項

- 一発で全部やらせる
- 長文で考えさせる
- 不要な履歴を持たせる
- 「とりあえず全部」依頼

---

## 8. 依頼テンプレ

```
目的：〇〇を修正したい
対象：該当コードのみ
出力：差分のみ / コードのみ
制約：最小コード・説明不要
```

---

## 9. 一文ルール

> **短く・分けて・軽く投げる**

---

## 10. Claude切り替え条件

以下は Claude に回す：

- 200行以上のコード
- 複数ファイル編集
- 設計変更
- バグ原因調査

---

## 🫧 本質

```
思考を減らす
  ↓
生成を減らす
  ↓
コストが下がる
```

> **節約 = 思考量の制御**
